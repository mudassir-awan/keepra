# Contributing to Keepra

Thanks for taking the time to contribute! This repo is the public community hub for Keepra - the source code is proprietary, but there are many meaningful ways to help.

---

## How you can help

### Bug reports
The most valuable contribution is a clear, reproducible bug report.

1. **Search first** - check [existing issues](https://github.com/keepraapp/keepra/issues) to avoid duplicates
2. **Use the bug template** when opening a new issue
3. **Include**: platform (Web / Windows / Android), browser/OS version, steps to reproduce, what you expected vs. what happened
4. **Attach screenshots or screen recordings** when relevant

### Feature requests
1. Check the [roadmap](docs/ROADMAP.md) - it may already be planned
2. Use the **Feature Request** issue template
3. Describe the problem you're solving, not just the solution - this helps us understand the use case

### Documentation
- Typos, unclear instructions, or missing steps in the guides below are fair game
- Submit corrections as issues (or if you have suggestions for specific rewording, paste it in the issue)

### Testing
- Install the latest release and try it on your platform
- Report regressions or edge cases you discover

---

## What we don't accept here

- Pull requests (source code is not in this repo)
- Requests to open-source the codebase
- Off-topic discussions unrelated to Keepra

---

## Issue etiquette

- One issue per bug / feature - don't bundle unrelated things
- Respond to maintainer questions within 14 days or the issue may be closed
- Be specific and constructive - vague reports are hard to act on

---

## Community standards

Be respectful. We follow a simple rule: treat contributors the way you'd want to be treated. Issues or comments that are abusive or off-topic will be closed.
